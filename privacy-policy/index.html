LexiTale 隱私權政策

生效日期：2026 年 7 月 13 日

LexiTale（以下稱「我們」）致力於保護您的隱私。本隱私權政策說明 LexiTale Android App 如何收集、使用、同步及保護您的資訊。

1. 資料收集

  • 本機學習資料：我們會在您的裝置上儲存學習進度，例如已學單字、收藏、隱藏狀態、複習日期、FSRS 複習參數及自訂單字。這些資料安全儲存在 App 的本地 SQLite 資料庫。
  • 登入資料：App 使用 Google 登入並透過 Firebase Authentication 處理登入，以識別使用者並取得基本登入資訊（如電子郵件或顯示名稱）。
  • 雲端同步資料：為防範資料丟失並提供進度保存，登入後您的學習進度會透過 Cloud Firestore 雲端資料庫保存。
  • 去識別化意見資料：若您使用 App 內回報功能，我們會保存回報類型、選填說明、必要的字卡 ID／內容包／排序參照，以及 App 版本、裝置型號與作業系統、語言、登入狀態和畫面位置等診斷資訊。回報不儲存 Firebase UID、姓名、信箱、截圖或整張字卡內容快照；疑似信箱與電話號碼會自動遮蔽。請勿在自由文字填入個人資料。
  • 外部字典查詢：當您新增自訂單字時，App 會透過 API 檢索字典釋義，不包含個人可識別資訊。

2. 資料使用

  • 您的學習資料僅用於提供核心單字記憶複習、自適應排程、進度統計與雲端還原同步。
  • 我們不出售任何使用者資料，也不會將您的學習歷史用於任何商業廣告追蹤。
  • 去識別化意見資料僅用於找出、排序與修正內容或產品問題。
  • 目前散布的 3.0.43+346 不包含第三方廣告或付款卡片處理；廣告版候選的資料處理另列於下方。

3. 資料保護

  • 本機資料嚴格限制在應用程式專屬 SQLite 沙盒中。
  • 雲端通訊使用安全傳輸加密，並依據 Firestore 資料庫安全規則限定個人僅能讀寫自己綁定 UID 的資料。
  • 核准的標準字卡圖片與音訊會視需要透過 Firebase Storage/CDN 下載，完成後保存在本機快取；每個級別另保留少量離線啟動素材。這些公開學習資產不含個人學習紀錄。

4. 權限說明

  • 網路存取 (INTERNET)：用於 Google 登入、進度同步、回報或客服資料傳輸、外部字典查詢及下載核准的字卡圖片與音訊；廣告版候選另用於廣告與彙總量測。
  • 震動控制 (VIBRATE)：用於在學習評分時提供流暢精緻的觸覺震動回饋。
  • 通知權限 (POST_NOTIFICATIONS)：僅用於您開啟的本地學習提醒。
  • 開機完成事件 (RECEIVE_BOOT_COMPLETED)：僅用於裝置重啟或 App 更新後重新排程已開啟的本地提醒。

5. 資料刪除

您可以在 App 的設定頁直接選擇「刪除帳號」，或使用公開的 帳號刪除說明網頁 送出申請。App 會刪除 Cloud Firestore 中的雲端學習進度與統計資料、舊版以帳號路徑保存的回報截圖附件、裝置本機 SQLite 學習資料與高品質圖片快取，並嘗試刪除 LexiTale 的 Firebase Authentication 登入帳號。舊版原始回報會完成一次性去識別化並刪除聯絡資料、附件參照與完整字卡快照；原始文件之後由管理端刪除。長期保留的去識別化意見紀錄不含帳號識別碼，可能作為產品與內容修正證據。若 Firebase 要求近期登入或 App 無法完成登入帳號刪除，您仍可透過公開帳號刪除頁或客服信箱要求我們完成帳號與相關雲端資料刪除。

LexiTale Privacy Policy

Effective Date: July 13, 2026

LexiTale ("we", "us", or "our") is dedicated to protecting your privacy. This policy describes how the LexiTale Android App collects, uses, syncs, and protects your information.

1. Data Collection

  • Local Learning Data: We save your learning progress (such as mastered words, bookmarks, hidden states, intervals, FSRS parameters, and custom words) securely in a local SQLite database.
  • Sign-In Data: The App uses Google Sign-In managed by Firebase Authentication to identify users and retrieve basic profile info (email and display name).
  • Cloud Synchronization: To prevent data loss, your progress is backed up securely in our Cloud Firestore database.
  • Deidentified Feedback: In-app feedback stores its type, optional details, the necessary card ID/pack/rank reference, and diagnostics such as App version, device model and OS, language, sign-in state, and screen location. It does not store a Firebase UID, name, email, screenshot, or full card-content snapshot. Likely email addresses and phone numbers are automatically redacted. Do not enter personal information in free text.
  • Dictionary Services: Adding custom words queries third-party dictionary APIs for definitions. No personally identifiable data is sent.

2. Data Usage

  • Your learning progress is strictly used to deliver card scheduling (FSRS), bookmarks, learning stats, and secure cloud sync/restoration.
  • We never sell your data, nor do we track your learning history for targeted commercial advertising.
  • Deidentified feedback is used only to find, prioritize, and fix content or product issues.
  • The currently distributed 3.0.43+346 does not process payments or serve third-party ads. The ads-enabled candidate is described below.

3. Data Security

  • Local data is isolated securely in the sandbox SQLite database of your Android device.
  • Cloud interactions are encrypted and restricted by Firebase Firestore security rules, allowing only authorized UID owners to read or write their own documents.
  • Approved standard card images and audio are downloaded on demand through Firebase Storage/CDN and cached locally after completion. A small offline starter set is bundled for each level. These public learning assets do not contain personal learning records.

4. Permissions

  • INTERNET: Required for Google Sign-In, cloud backups, report/support data transfer, dictionary API queries, and approved card-image and audio downloads. The ads-enabled candidate also uses it for advertising and aggregate measurement.
  • VIBRATE: Used to trigger subtle haptic vibrations during card rating to enhance UX.
  • POST_NOTIFICATIONS: Used only for local study reminders you enable.
  • RECEIVE_BOOT_COMPLETED: Used only to reschedule enabled local study reminders after device restart or App update.

5. Data Deletion

You can select "Delete Account" in the App Settings sheet or submit a request on our Account Deletion Web Page. The App deletes your Cloud Firestore learning progress and statistics, legacy report screenshot attachments stored under the account path, local SQLite learning data, and high-quality image caches, and attempts to delete your LexiTale Firebase Authentication login account. Legacy raw reports undergo a one-time deidentification that removes contact data, attachment references, and full card snapshots; administrators then delete the raw documents. Long-term deidentified feedback has no account identifier and may be retained as evidence for product and content fixes. If Firebase requires recent sign-in or the App cannot complete login-account deletion, you can still use the public deletion page or support email to request deletion of the account and associated cloud data.

6. Ads-enabled candidate data processing (not yet released)

A later ads-enabled version will use Google Mobile Ads for rewarded and low-frequency interstitial ads. The SDK may process approximate location inferred from IP, App interactions, diagnostics, and device identifiers for advertising, analytics, fraud prevention, security, and compliance. The App uses UMP for applicable consent and provides ad privacy options when required. Firebase Analytics records only aggregate study-flow and ad outcomes; it does not send a Firebase UID, name, email, card text, or free-form input.

7. Contact Us

If you have any questions or feedback regarding this policy, contact us at: lexitale.support@gmail.com

LexiTale プライバシーポリシー

発効日：2026年7月13日

LexiTale（以下「私たち」）は、ユーザーのプライバシーの保護に努めています。本ポリシーでは、LexiTale Android アプリがユーザーの情報をどのように収集、使用、同期、および保護するかについて説明します。

1. 情報の収集

  • ローカル学習データ：学習進度（学習単語、ブックマーク、アーカイブ状態、FSRS 復習パラメーター、カスタム単語など）は、端末のローカル SQLite データベースに安全に保存されます。
  • ログイン情報：アプリは Google ログインを使用し、Firebase Authentication を介してユーザーを識別し、基本プロファイル（メールアドレス、表示名）を取得します。
  • クラウド同期データ：データの紛失を防ぐため、ログイン後は Cloud Firestore に学習履歴が自動的にバックアップされます。
  • 匿名化フィードバック：アプリ内フィードバックには、種類、任意の説明、必要なカード ID／パック／順位参照、アプリのバージョン、端末モデルと OS、言語、ログイン状態、画面位置などの診断情報を保存します。Firebase UID、氏名、メール、スクリーンショット、カード内容全体のスナップショットは保存しません。メールアドレスや電話番号らしい文字列は自動で伏せ字にします。自由記述に個人情報を入力しないでください。

2. 情報の使用

  • 学習データは、単語カードの復習スケジュール調整、学習統計、およびクラウド復元同期の提供のみに使用されます。
  • 私たちはユーザーデータを第三者に販売せず、学習履歴を広告ターゲティングに利用しません。
  • 匿名化フィードバックは、コンテンツまたは製品の問題の発見、優先順位付け、修正にのみ使用します。
  • 現在配布中の 3.0.43+346 には第三者広告や決済処理は含まれていません。広告版候補のデータ処理は下記に記載します。

3. 情報の保護

  • 端末内のデータは、アプリ専用の SQLite サンドボックスに隔離保護されます。
  • クラウドとの通信は暗号化され、Firestore の安全規則によって本人のみがデータにアクセスできるように制限されています。
  • 承認済みの標準カード画像と音声は Firebase Storage/CDN から必要に応じてダウンロードされ、完了後は端末にキャッシュされます。各レベルには少量のオフライン開始用素材を同梱します。これらの公開学習アセットには個人の学習履歴は含まれません。

4. アプリ権限の説明

  • インターネットアクセス (INTERNET)：Google ログイン、同期処理、報告・サポートデータ送信、外部辞書 API 照会、承認済みカード画像・音声のダウンロードに必要です。広告版候補では広告と集計測定にも使用します。
  • バイブレーション制御 (VIBRATE)：カードの評価時、学習体験向上のための心地よい触覚フィードバックを提供します。
  • 通知権限 (POST_NOTIFICATIONS)：ユーザーが有効にしたローカル学習リマインダーにのみ使用します。
  • 起動完了イベント (RECEIVE_BOOT_COMPLETED)：端末再起動またはアプリ更新後、有効なローカル学習リマインダーを再設定するためにのみ使用します。

5. データの削除

アプリの「アカウント削除」ボタン、または アカウント削除申請ページ から手続きを行えます。アプリは Cloud Firestore の学習進度と統計、アカウントのパスに保存された旧版の報告スクリーンショット添付、端末の SQLite 学習データ、高品質画像キャッシュを削除し、LexiTale の Firebase Authentication ログインアカウントの削除を試みます。旧版の生の報告は一度限りの匿名化処理で連絡先、添付参照、カード全体のスナップショットを除去し、その後管理者が元の文書を削除します。長期保持する匿名化フィードバックにはアカウント識別子がなく、製品・コンテンツ修正の証拠として保持される場合があります。Firebase が最近のログインを要求する場合、またはアプリ内でログインアカウント削除を完了できない場合でも、公開削除ページまたはサポートメールからアカウントと関連クラウドデータの削除を申請できます。

6. 広告版候補のデータ処理（未公開）

後続の広告版では Google Mobile Ads を使用し、リワード広告と低頻度のインタースティシャル広告を表示します。SDK は IP から推定した大まかな位置、アプリ操作、診断情報、端末識別子を広告、分析、不正防止、安全、法令遵守のために処理する場合があります。アプリは UMP で必要な同意を管理し、必要な場合は広告のプライバシー設定を提供します。Firebase Analytics は集計した学習フローと広告結果のみを記録し、Firebase UID、氏名、メール、カード本文、自由入力は送信しません。

  • 비식별 피드백: 앱 내 피드백에는 유형, 선택 설명, 필요한 카드 ID／팩／순위 참조, 앱 버전, 기기 모델과 OS, 언어, 로그인 상태, 화면 위치 등의 진단 정보를 저장합니다. Firebase UID, 이름, 이메일, 스크린샷 또는 전체 카드 내용 스냅샷은 저장하지 않습니다. 이메일 주소나 전화번호로 보이는 문자열은 자동으로 가립니다. 자유 입력란에 개인정보를 넣지 마세요.

2. 정보 사용

  • 사용자의 학습 데이터는 단어 카드 학습 최적화(FSRS), 북마크, 학습 통계 및 안전한 클라우드 진도 복원 기능에만 제한적으로 사용됩니다.
  • 당사는 사용자의 어떠한 데이터도 판매하지 않으며, 학습 이력을 타겟 광고 등에 활용하지 않습니다.
  • 비식별 피드백은 콘텐츠 또는 제품 문제를 찾고 우선순위를 정해 수정하는 데만 사용합니다.
  • 현재 배포 중인 3.0.43+346은 제3자 광고를 제공하거나 결제 데이터를 처리하지 않습니다. 광고 버전 후보의 데이터 처리는 아래에 설명합니다.

3. 정보 보호

  • 기기 내부의 데이터는 앱 전용 SQLite 샌드박스로 격리되어 안전하게 보호됩니다.
  • 클라우드와의 모든 통신은 보안 암호화되며, Firestore 보안 규칙에 의거하여 승인된 UID 본인만 데이터에 액세스할 수 있습니다.
  • 승인된 표준 카드 이미지와 오디오는 Firebase Storage/CDN에서 필요할 때 다운로드되며 완료 후 로컬에 캐시됩니다. 각 수준에는 소량의 오프라인 시작용 미디어가 포함됩니다. 이러한 공개 학습 자산에는 개인 학습 기록이 포함되지 않습니다.

4. 권한 설명

  • 네트워크 액세스 (INTERNET): Google 로그인, 클라우드 동기화, 신고/지원 데이터 전송, 외부 사전 API 조회, 승인된 카드 이미지와 오디오 다운로드에 사용됩니다. 광고 버전 후보에서는 광고와 집계 측정에도 사용됩니다.
  • 진동 제어 (VIBRATE): 카드 평가 시 부드러운 햅틱 진동 피드백을 제공하여 학습 경험을 향상시킵니다.
  • 알림 권한 (POST_NOTIFICATIONS): 사용자가 켠 로컬 학습 알림에만 사용됩니다.
  • 부팅 완료 이벤트 (RECEIVE_BOOT_COMPLETED): 기기 재시작 또는 앱 업데이트 후 켜져 있는 로컬 학습 알림을 다시 예약하는 데만 사용됩니다.

5. 데이터 삭제

앱 내 설정 메뉴의 "회원탈퇴"를 선택하거나 외부 회원탈퇴 신청 페이지를 통해 언제든지 요청할 수 있습니다. 앱은 Cloud Firestore 학습 기록과 통계, 계정 경로에 저장된 이전 버전 신고 스크린샷 첨부, 기기의 SQLite 학습 데이터 및 고화질 이미지 캐시를 삭제하고 LexiTale Firebase Authentication 로그인 계정 삭제를 시도합니다. 이전 버전의 원본 신고는 일회성 비식별화 과정에서 연락처, 첨부 참조, 전체 카드 스냅샷을 제거한 뒤 관리자가 원본 문서를 삭제합니다. 장기 보관되는 비식별 피드백에는 계정 식별자가 없으므로 제품 및 콘텐츠 수정 근거로 보관될 수 있습니다. Firebase가 최근 로그인을 요구하거나 앱에서 로그인 계정 삭제를 완료할 수 없는 경우에도 공개 삭제 페이지 또는 지원 이메일을 통해 계정과 관련 클라우드 데이터 삭제를 요청할 수 있습니다.

6. 광고 버전 후보의 데이터 처리(아직 출시되지 않음)

후속 광고 버전은 Google Mobile Ads로 보상형 광고와 저빈도 전면 광고를 표시합니다. SDK는 IP로 추정한 대략적인 위치, 앱 상호작용, 진단 정보, 기기 식별자를 광고, 분석, 부정 방지, 보안 및 법적 준수에 사용할 수 있습니다. 앱은 UMP로 필요한 동의를 관리하고 필요할 때 광고 개인정보 보호 옵션을 제공합니다. Firebase Analytics는 집계된 학습 흐름과 광고 결과만 기록하며 Firebase UID, 이름, 이메일, 카드 본문 또는 자유 입력은 전송하지 않습니다.
